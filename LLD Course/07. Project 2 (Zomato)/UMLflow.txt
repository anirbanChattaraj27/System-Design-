PLACING ORDER LLD:

MODEL Directory:
    this is simple class, which has getters and setters. It has set of varibles, which can make setter and getter. we can represent model class directly in DB in form of tables 

    Class Restaurent: 
        int id
        String name
        String address
        list<MenuItem>

    class MenuItem:
        int Code
        String name
        double price

Restaurent has-a MenuItem (one-to-many relationship) ---> this is a composition relationship. ---> MenuItem can not exist without Restaurent --> use dimond shape to connect classes
 


Manager Directory: (Singleton)

    RestaurentManager:
        here we perform CRUD(create-read-update-delete) operation on Restaurent. bcz there will be multiple Restaurent on zomato, and Restaurent keep adding and removing
        List<Restaurent>
        CRUD operation
        SearchByLocation (loc)

RestaurentManager has-a Restaurent (one-to-many relationship) ---> this is Aggregation. ---> Restaurent can exist without RestaurentManager ---> Dimond shape to connect classes.



MODEL Directory:

    class User:

        Cart cart
    
    class Cart:

ORDER FACTORY Directory:

    this will use to create Orders
    Here we will use FACTORY Design Pattern

    interface class OrderFactory:
        createOrder() ---> abstract method

    now we will create classes that will @Override this interface...

    class NowOrder:
        @Override createOrder
    class ScheduleOrder:
        @Override createOrder
